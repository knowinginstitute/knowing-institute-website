Knowing Protocol

First Draft, March 2021.

Abstract

The Knowing Protocol is a semantic network and a blockchain of ideas, which offers an alternative solution for societal problems coming from diversive sense-making processes. Instead of trying to control minds top-down in an effort to create a stable society, it offers a way to bootstrap democracy by being a reference point when communicating ideas, and a schema for understaning any reality experienced by humans. This also offers a solution to future problems when Articifial Intelligence will outperform human cognitive abilities, giving us transparence and traceability.

The Semantic Network differs from previous semantic networks attempts in several ways, most notably by introducting the domains Sense, Mind and Being, and adding forms for state-handling, which gives the way for creating abstract representations of ehtical systems. It also holds the ability to give concepts grounded meanings, which is an important function when creating AI-systems understanding human language.

The Blockchain consists of a primary layer, which holds concepts and ideas agreed upon by all humankind, including shared values and concepts related to human rights. This layer can only change when a two-thirds of token-holders agree. Secondary layers can be any science knowledge or ideaology wich doesn't pass the way to be included in the primary layer. Third-level layers are outside the distributed ledger, and holds any fictional or real human/machine representation which may include unique memories, knowledge, oppinions etc. All layers offers means of tracability.

The Semantic Network definition can be summarized in this prototype:

  1. Reality is; physical objects with properties
  2. and abstract ideas,
  3. represented by symbols
  4. that are categorized,
  5. have relations,
  6. and can interact with each other
  7. within restrictions,
  8. interpreted in contexts by agents
  9. having bias and ethics,
  10. giving them attributes and meaning.

Background

There have been several attempts for creating abstract representation of human knowledge through a semantic network, which is a knowledge base representing semantic relations between concepts in a network useful for solving problems by artificial reasoning.

Some notable examples are Wordnet, Conceptnet, CyCorp and a number of proprietary graphs from Google, Facebook etc. While they all are add value in their own specialized field, neither of them offer an exhausted representation of how interconnected agents creates meaning of a reality comprising both real-world objects and emerging ideas. At best they offer only a single single-dimensional representation of something not really well-specified.

Why do we need this?

There are several reasons why we need a single model for representing individual and collective sense-making, accumulate knowledge, and enable efficient information transfer between humans and machines.

  • Having a shared reference point, so we know what we are talking about. This will help us detect how our ideas relate to each-other and how the concepts we are using for describing reality changes not only depending on who is using them, but even in what context we are using them alone. Once we have this reference point, we can focus on real discussions, and not only debating how to describe the reality.
  • Give more people the ability to understand complex problems using powerful Articifial Intelligence based on this shared knowledge of world-, semantic- and agent-centric objects. That kind of AI would be an instrument for creating a better democracy bottom-up rather than having a government using AI for controlling the masses top-down.
  • Create a new kind of Articifial Intelligence combining Deep Learning and Symbolic Reasoning, addressing challenges of reasoning, hierarchical representations and learning transferring. A huge benefit would be to get traceability of how AI models the reality and makes decisions which is very dificult in pure statistical deep learning models.

A new approach

The Knowing Protocol have a what-works-works approach, which means whatever works in practice might also work in theory. Even if the process of creating the model includes extended research in multiple fields, including linguistics, semantics, psychology and philosophy, it has no ambision taking on a more philosophical or academic approach than necessary. It has emerged from a learn-by-prototyping process after finding there were no existing semantic framworks covering the full extent of human knowledge including individual and collective sense-making. Given that, it is still a work in progress and will be continuesly developed as need for new qualites emerges.

The Protocol uses these existing frameworks as starting points,

  • PropBank
  • Roget Thesaurus
  • Blockchain

The primary take-aways from the Knowing Protocol are:

  • Connecting mentioned frameworks in a single model
  • Introducing three domains; Sense, Mind and Being
  • Introducing a method for grounding meaning using semantic frameworks for state, cause-effect, time and storytelling
  • Introducing blockchain of ideas, including layers, tokens and support for transaction fingerprints
Problems with Symbolic Ai

The Protocol permit the use of Symbolic Ai, which is often criticized for:

  • Being monotonic, ie it is difficult to change beliefs once they are encoded in a rules engine. Knowing Protocol tries to solve that problem by Semantic Operations.
  • Not having any real mening, ie symbols are not really grounded and it’s difficult to relate them to to other forms of meaning. Knowing Protocol tries to solve that problem the Three Domains and the breakdown of Concepts into States.
  • Force machines to communicate through natural language although it is probably not optimal for a machine-to-machine communication. The purpose of the Knowing Protocol is to create shared semantic networks both humans and machines can understand and use, so the basis for the procotol is the opposite: it's necessary for any communication describing a human-centric reality to use natural language, including machine-to-machine communication in that domain.

The Model

The Knowing Protocol describes how to draw a knowledge graph comprising vertices representing semantic symbols and edges representing semantic relations between these symbols.

Prototype

The definition of this semantic network is stated in this prototype:

  1. Reality is; physical objects with properties
    Our basic understanding of the physical world is it consists of objects with mesurable properties, which may be the result of a physical process.
  2. and abstract ideas,
    Ideas are mental objects or mental states which an agent uses when making sense about phsyical objects or ideas.
  3. represented by symbols
    We create an abstract representation of the language we use when interpreting objects and ideas.
  4. that are categorized,
    We categorize these symbols so they make more sense to us. For example, we call some of them Nouns, and some of them a Color. Or mammuls.
  5. have relations,
    We relate these symbols with each-other for understanding concepts, cause-effect, meaning, etc.
  6. and can interact with each other
    Symbols (ie physical objects and ideas) can interact with each-other in different ways.
  7. within restrictions,
    Interactions between symbols can only happen within given restrictions; physical, social, psyhological, etc.
  8. interpreted in contexts by agents
    Whenever a symbol, or groups of symbols, are understood by an agent (human or machine), that's happening within a context, like a conversation, culture, ideology, etc.
  9. having bias and ethics,
    Agents have bias and an ethical system, which happens when symbols and relations in the model are compared and valued, especially around case-effect and choice.
  10. giving them attributes and meaning.
    When agents interpret symbols in a sense-making process, based on context, bias and ethics, they give symbols attributes and meaning.

Domains

All symbols are conceptual, which means they all represents mental objects or mental states used by agents when making sense of their reality. They are then divided into three domains based on how they are used in that process; based on how humans experience their reality and where their attention is in that moment.

  • Senses Domain
    Answers the question: What Is? Attention is close to our senses.
  • Mind Domain
    Answers the question: What Reveils? Attention is more in our mind.
  • Being Domain
    Answers the question: What Ought? Attention is more settled in being.

Another way of naming the domains could be hand-thought-gut.

Senses Domain

Semantic symbols and relations in the Senses domain are close to real-world, they often represent a physical object or processes with measurable properties. These symbols gerenally don't need reasoning or abstract thinking.

  • Symbols in this domain raises from the question: What is?
  • They are close to what we as humans can experience through senses
  • The appear to us when we have our attention outside ourselves.
  • Example: This flower is red. Universe is big. Chickens lay eggs.

Mind Domain

Symbols in the mind domain are more abstract and related to thinking, reasoning, and concepts generated by reflection or a thought process.

  • Symbols raises from the question: What Reveils?
  • They appear to us when our attention to some extent needs to be inside our minds.
  • Most of the symbols, relation and categories found in the semantic network are here.
  • Example: Some flowers are red. Universe has a lot of undiscovered black holes. Chickens learn to walk before they can fly.
  • There are subdomains illustrating the complexity of ideas

Being Domain

Symbols in the being domain comes from agent-centric processes, how an agent attribute symbols in previous domains, and are related to a preferences, values and ethical system.

  • Symbol raises from the question: What ought?
  • They appear to us when we have our attention deeper inside ourselves
  • They come from our experience and beliefs
  • I like red flowers. Universe seams terrifying. I want a chicken for president.

A complete reference of symbols are found in the Appendix.

State handling

The protocol permits to group symbols into states and then relate this states with each-other for describing Change. This is usable in many ways;

  • Grounded Meaning
    We can define a concept as the difference between different states, and by that add more ground to it's meaning. [Explain Grounded Meaning and agent-centric meaning.]
  • Relation to time
    We can connect states to time-stamps and order events after when the occure in any given timeline.
  • Cause-Effect
    States can capture cause-effect-relations.
  • Storytelling
    Storytelling is a fundamental way of understanding the reality for humans. It's also a way of grounding the knowledge for Ai. Without storytelling the knowledge dont' have the same meaning.
  • Ethical System
    We can use states for attaching values in the Being-domain to changes between states, ie an agent can have a preference connected to any change, which is the base for an ethical system.

The Protocol uses states instead of many relations found in other frameworks.

Operations

The Protocole permits a set of semantic network operations which are important for making the graph an agile representation of human knowledge, where concepts sometimes change definitions depending on context and ethical systems, and the ability to merge module-based knowledge and concepts into largers sets.

Layers

The Semantic Network han in theory hold unlimited number of layers, where each layer is a set of nodes representing specific knowledge or values. These layers can be merged together when using to network for performing any external task.

The Blockchain protocol defines a primary layer holding general knowledge agreed upon by most of humankind. Anyone can the define a secondary layer based on a specific scientific knowledge or ideology, and load that ontop on the primary layer when for instance performing a task of analysing a text. The result of that task will then ba based on the sum of all layers currently loaded in the semantic network.

Type-3 layers are generally representing fictional or real humans knowledge and values.

Displacement

Displacement of concepts are an important quality in the semantic network. It describes how a concept can change definition by making a copy of itself and place that copy in a separate context, as a way of alter it's definition in a particual use-case.

The Blockchain

The "Blockchain of Ideas" are defined as a distributed ledger system, utulizing blockchain technology.

Layers

The Blockchain holds three different types of layers:

  • Primary Layer holding concepts and ideas agreed upon by all humankind, including shared values and concepts related to human rights. This layer can only change when a two-thirds of token-holders agree
  • Secondary Layers holding any science knowledge or ideaology wich doesn't pass the way to be included in the primary layer. These layers are created open-source and driven in community-style development.
  • Type-3 Layers holding knowledge, memories, oppinions, etichal systems connected to fictional or real human/machine representations. These layers lays outside the distributed ledger and can be used as layers in any application. They also lack the tracability of transactions.

Tokens

Blockchain tokens called Knowens are used for voting on changes in the primary layer. Owners of Knowens will be institutions representing different cultures.

Fingerprint and traceability

Every future statement generated by a Artificial Intelligence using the Knowing Protocol, will have a fingerprint which makes it possible to trace what ideas and concept, and what definitions, is has used to come to that conclusions. This will make it possible to counter-act frogery of mind, and control by AI top-down.

Implementation

The implementation of the Knowing Protocol will be orhestered by Knowing Institute in a module-based rapid prototyping style project, including the following blocks:

  • Organisation, including:
    • Advisory board
    • Network of ambassadors
  • The Protocol, including:
    • Involve experts from different fields to add suggestions
    • Improve the protocol
  • Technology, including:
    • Finish development of the open-source distributed graph ledger
    • Finish development of wiki-knowing
    • Develop fingerprint technology for future transactions based on AI
  • Primary Layer, including:
    • Converting existing sources of inforamtion to primary layer
    • Distribute Knowens-tokens to instiutions
    • Establish a common ground
  • Secondary and Third Layers, including:
    • Invite people to create secondary candidates on the wiki-platform
    • Develop and offer SDK for the user of Layer-3 in any application

We respect it will take time to fill this semantic network with content.
