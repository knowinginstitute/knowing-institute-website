Knowing Protocol

First Draft, February 2021.

“Scientific method is just the first step in creating what can be considered as truth. It still starts and ends with language.”

Introduction

The Knowing Protocol offers a common ground for understanding a world created by interconnected agents. It can be used as a framework for sense-making, or as a shared protocol for accurately transfer meaning between humans and machines.

The Protocol defines a Semantic Network which can fully represent any reality experienced by humans. It differs from previous semantic networks in several ways, most notably in the assumption that any experienced reality is best understood from three domains; physical, semantic and agent-centric, and it also has a method for giving concepts and ideas grounded meanings.

The Protocol can be summarized in this prototype:

  1. Reality is; physical objects with properties
  2. and abstract ideas,
  3. represented by symbols
  4. that are categorized,
  5. have relations,
  6. and can interact with each other
  7. within restrictions,
  8. interpreted in contexts by agents
  9. having bias and ethics,
  10. giving them attributes and meaning.

Background

There have been several attempts for creating abstract representation of human knowledge through a semantic network, which is a knowledge base representing semantic relations between concepts in a network useful for solving problems by artificial reasoning.

Some notable examples are CyCorp, Wordnet, Conceptnet and a number of proprietary graphs from Google, Facebook etc. While they all are add value in their own specialized field, neither of them offer an exhausted representation of how interconnected agents creates meaning of a reality comprising both real-world objects and emerging ideas. At best they offer only a single single-dimensional representation of something not really well-specified.

Why do we need this?

There are several reasons why we need a single model for representing individual and collective sense-making, accumulate knowledge, and enable efficient information transfer between humans and machines.

  • Having a shared reference point, so we know what we are talking about. This will help us detect how our ideas relate to each-other and how the concepts we are using for describing reality changes not only depending on who is using them, but even in what context we are using them alone. Once we have this reference point, we can focus on real discussions, and not only debating how to describe the reality.
  • Give more people the ability to understand complex problems using powerful Articifial Intelligence based on this shared knowledge of world-, semantic- and agent-centric objects. That kind of AI would be an instrument for creating a better democracy bottom-up rather than having a government using AI for controlling the masses top-down.
  • Create a new kind of Articifial Intelligence combining Deep Learning and Symbolic Reasoning, addressing challenges of reasoning, hierarchical representations and learning transferring. A huge benefit would be to get traceability of how AI makes decisions and create models the reality.

A new approach

The Knowing Protocol has emerged from a learn-by-prototyping process while creating a conversational interface, and a gradual discovery no available semantic network was really useful. In addition, an ambition to include individual and collective sense-making as an important dimensions in the model, left no option than building a new model from the ground-up and then look for ways to transfer knowledge from other models into this one. The process of creating the model includes extended research in multiple fields, including linguistics, semantics, psychology, as well as paying much attention to how thoughts are formed and used in my own comprehension and sense-making.

The Protocol uses these existing frameworks as starting points,

  • PropBank and AMR
  • Roget Classification
  • Wordnet
  • Primary

The primary take-aways from the Knowing Protocol are:

  • Connecting mentioned frameworks in a single model
  • Introducing three domains; Physical, Semantic and Agent-centric
  • Introducing a method for grounding meaning using semantic frameworks for state, cause-effect, time and storytelling

Semantic Research

While there’s an extended research in the semantics-field and we know some parts in this model will be open for critique from different niches in the semantic research, this model has three primary objectives:

  • It is a single framework covering all aspects of semantic representation, which means there are some compromises and simplifications which might not be necessary in a pure theoretical framework.
  • Knowing Protocol is designed to be useful for real world applications and not only a theoretical framework.
  • Knowing Protocol takes advantage of being pragmatic and emerged from learn-by-prototyping dynamics.
Problems with Symbolic Ai

The Protocol permit the use of Symbolic Ai, which is often criticized for:

  • Being monotonic, ie it is difficult to change beliefs once they are encoded in a rules engine. Knowing Protocol tries to solve that problem by Semantic Operations, and Layered Networks (se below).
  • Not having any real mening, ie symbols are not really grounded and it’s difficult to relate them to to other forms of meaning. Knowing Protocol tries to solve that problem the Three Domains and the breakdown of Concepts into States.
  • Force machines to communicate through natural language although it is probably not optimal for a machine-to-machine communication. The purpose of the Knowing Protocol is to create shared semantic networks both humans and machines can understand and use.

The Model

The Knowing Protocol describes how to draw a graph comprising vertices representing semantic symbols and edges representing semantic relations between these symbols. It also includes

  • a predefined set of actions, which represents how the symbols can interact with each-other
  • a predefined set of property-groups, which represents measurable properties on physical objects
  • a predefined set of attribute-groups, which represents agent-centric ways of describing the experience of objects and ideas

Domains

In order to create semantic network fully representing human knowledge, on contrast to existing semantic networks, we use three different classes for abstraction of language:

  • Physical Domain, holding objects from the physical reality, with measurable properties
  • Semantic Domain, holding objects from the semantic representation of both physical and abstract ideas
  • Agent-Centric Domain, holding objects coming from an agent’s experience and sense-making of previous domains

In effect we let the model represent an experienced reality rather than an objective one, and we also include the process of collective sense-making as well as methods for representing ethical systems.

Prototype

The protocol specifies how to use a semantic network for describing reality, and it is best summarized in this prototype:

  1. Reality is; physical objects with properties
    Physical Our basic understanding of the physical world is it consists of objects with mesurable properties, which may be the result of a physical process.
  2. and abstract ideas,
    Agent-centric Ideas are mental objects or mental states which an agent uses when making sense about phsyical objects or ideas.
  3. represented by symbols
    Semantic We create an abstract representation of the language we use when interpreting objects and ideas.
  4. that are categorized,
    Semantic We categorize these symbols so they make more sense to us. For example, we call some of them Nouns, and some of them a Color.
  5. have relations,
    Physical Semantic Agent-centric We relate these symbols with each-other to understand cause-effect, .
  6. and can interact with each other
    Physical Semantic Agent-centric Both the symbols themselves (semantics), and what the symbols represents (objects), can interact with each-other in different ways.
  7. within restrictions,
    Physical Semantic Agent-centric Interactions between symbols can only happen within given restrictions; physical, social, psyhological, etc.
  8. interpreted in contexts by agents
    Semantic Agent-centric Whenever a symbol, or groups of symbols, are understood by an agent (human or machine), that's happening within a context, like a conversation, culture, ideology, etc.
  9. having bias and ethics,
    Agent-centric Agents have bias and an ethical system, which happens when symbols and relations in the model are compared and valued, especially around case-effect and choice.
  10. giving them attributes and meaning.
    Semantic Agent-centric When agents interpret symbols in a sense-making process, based on context, bias and ethics, they give symbols attributes and meaning.

Symbols

  • Physical objects, like a Stone or a Roaster. One way of understanding the physical world is to say it consists of objects with properties, where properties are defined as:
    Properties are measurable and might be the result of a physical process.
  • Semantic objects, like Noun or Action. Semantic objects are used when defining physical and agent-centric objects. by when grouping, relating, or making operations in the
  • Agent-centric objects, like Beautiful or Democracy. Abstract ideas are equally important as physical objects in order for agents to understand a shared reality-space. Ideas are defined as:
    Mental objects or mental states used by an agent when making sense about something.

As already mentioned, these domains both overlap and affect each-other when an agent create meaning of reality, but the division is good enough for us to be able to create a full semantic network.

Physical Domain

Since physcial objects are measuarable, we benefit from having them in this separate category. One way of looking at it is; whatever object is not subject to a agent-centric process is a physical object. A green apple is physical. A beautiful apple is not (the apple still is, but the beautiful-aspect of it is not). A forbidden fruit in paradise is not a physical-only object either.
